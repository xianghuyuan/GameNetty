# MoveComponent 详细实现文档（自驱动架构）

## 📋 概述

`MoveComponent` 是 ET 框架中负责 **Unit 移动逻辑** 的核心组件，采用**目标驱动 + 自主寻路 + 定时器驱动**的异步移动机制。

**核心设计理念**：
- ✅ **自驱动**：组件内部自主管理移动状态，不依赖外部传递路径
- ✅ **目标驱动**：只需设置 `TargetPosition`，组件自动完成寻路和移动
- ✅ **逻辑锁机制**（MoveForbiddenCount）用于 AI 控制
- ✅ **自动寻路**：集成 `PathfindingManagerComponent`
- ✅ **异步通知**：完成时自动触发事件

| **AI 锁** | MoveForbiddenCount 计数器 |

## 📦 数据结构详解

### MoveComponent 核心字段

```csharp
[ComponentOf(typeof(Unit))]
public class MoveComponent: Entity, IAwake<int>, IDestroy
{
    // ========== 目标驱动（核心） ==========
    public float3? TargetPosition;             // 目标位置（null=无目标）
    
    // ========== AI 逻辑锁 ==========
    public int MoveForbiddenCount;              // 移动禁制计数器（0=可移动，>0=禁止移动）
    public bool CanMove => MoveForbiddenCount == 0 && TargetPosition != null;

    // ========== 速度控制 ==========
    public float Speed;                         // 移动速度（初始化时设置）

    // ========== 路径点系统（内部使用） ==========
    public List<float3> Targets;                // 当前路径点列表
    public int N;                               // 当前路径点索引

    // ========== 时间控制 ==========
    public long BeginTime;                      // 移动开始时间
    public long StartTime;                      // 当前段的开始时间
    public long NeedTime;                       // 当前段需要的时间

    // ========== 位置插值 ==========
    public float3 StartPos;                     // 当前段起始位置

    // ========== 转向控制 ==========
    public int TurnTime;                        // 转向平滑时间（100ms）
    public bool IsTurnHorizontal;               // 是否只水平转向
    public quaternion From;                     // 起始旋转
    public quaternion To;                       // 目标旋转

    // ========== 异步控制 ==========
    public ETTask<bool> tcs;                    // 完成通知任务
    public long MoveTimer;                      // 定时器 ID
}
```

### 关键属性说明

| 属性 | 类型 | 说明 |
|------|------|------|
| `TargetPosition` | float3? | **核心**：设置目标点，组件自动寻路移动 |
| `CanMove` | bool | 是否可以移动（锁检查 + 目标检查） |
| `PreTarget` | float3 | 上一个路径点 `Targets[N-1]` |
| `NextTarget` | float3 | 下一个路径点 `Targets[N]` |
| `FinalTarget` | float3 | 最终目标点 `Targets[Last]` |

## 🔄 核心流程

### 1. 自驱动移动流程

```csharp
// 外部调用：只需设置目标
moveComponent.SetTarget(targetPosition);
    ↓
1. 设置 TargetPosition = targetPosition
    ↓
2. 等待 Timer 回调（每帧）
    ↓
3. MoveForward() 检查：
    ├─ if (TargetPosition == null): return
    ├─ if (!CanMove): return（被锁定）
    └─ if (Targets.Count == 0): 自动寻路
        ├─ PathfindingManagerComponent.Find()
        └─ 填充 Targets 列表
    ↓
4. 执行位置插值移动
    ↓
5. 到达目标：
    ├─ if (距离目标 > 0.5f): 继续寻路
    └─ else: MoveFinish(true)
```

### 2. 每帧移动流程（Timer 回调）

```csharp
MoveTimer.Invoke()
    ↓
MoveForward(true)
    ↓
1. 检查 CanMove
    ├─ if (!CanMove): return
    │   ├─ MoveForbiddenCount > 0（被锁定）
    │   └─ TargetPosition == null（无目标）
    │
    ↓
2. 自动寻路（仅首次）
    if (Targets.Count == 0 && TargetPosition != null):
    ├─ PathfindingManagerComponent.Find()
    └─ 填充 Targets 列表
    ↓
3. 计算时间差：moveTime = ClientNow() - StartTime
    ↓
4. while (moveTime > 0) 循环：
    ├─ if (moveTime >= NeedTime):
    │   └─ 直接到达 NextTarget
    │
    ├─ else:
    │   ├─ pos = lerp(StartPos, NextTarget, amount)
    │   └─ rot = slerp(From, To, amount)
    │
    ├─ moveTime -= NeedTime
    │
    ├─ if (moveTime < 0): return（本帧时间用完）
    │
    ├─ if (N >= Targets.Count - 1):
    │   ├─ if (距离目标 > 0.5f):
    │   │   └─ 清空 Targets，继续寻路
    │   └─ else:
    │       └─ MoveFinish(true)
    │
    └─ SetNextTarget() - 准备下一段
```


```csharp

// 2. 执行移动插值
// 3. 到达后自动停止
```

### 3. 停止移动

```csharp
// 方法 1：清空目标
move.TargetPosition = null;

// 方法 2：使用 Stop 方法
move.Stop(true);  // 正常停止
move.Stop(false); // 强制停止
```

### 4. 传送（瞬间移动）

```csharp

// TargetPosition 会自动清空
```

### 5. 速度动态调整

```csharp
// 移动过程中改变速度
bool success = move.ChangeSpeed(8f);
if (!success)
{
    // 已经到达或速度无效
}
```

### 6. AI 锁控制（重要！）

```csharp
MoveComponent move = unit.GetComponent<MoveComponent>();

// 锁定移动（攻击、施法、眩晕等）
move.MoveForbiddenCount++;

// ... 执行某些逻辑 ...
// 注意：锁定时移动会暂停，但 TargetPosition 保持不变

// 解锁移动
move.MoveForbiddenCount--;

// 组件会自动恢复移动（如果 TargetPosition != null）
```

## 🎯 AI 集成方式

### 怪物 AI 使用示例

```csharp
public static async ETTask<bool> ExecuteAttackAsync(Unit unit, Unit target)
{
    MoveComponent move = unit.GetComponent<MoveComponent>();
    
    // 1. 加锁，清空目标（停止自主移动）
    move.MoveForbiddenCount++;
    move.TargetPosition = null;
    
    try
    {
        float distance = math.distance(unit.Position, target.Position);
        
        // 2. 如果距离过远，先移动
        if (distance > combat.AttackRange)
        {
            move.MoveForbiddenCount--;
            
            // 设置目标，让组件自主移动
            // 注意：MoveToTargetServerAsync 会持续监控距离
            return await MoveToTargetServerAsync(unit, target, combat.AttackRange * 0.8f);
        }
        
        // 3. 执行攻击
        await combat.Attack(target, false);
        return true;
    }
    finally
    {
        // 4. 解锁
        move.MoveForbiddenCount--;
    }
}

public static async ETTask<bool> MoveToTargetServerAsync(Unit unit, Unit target, float stopDistance)
{
    MoveComponent move = unit.GetComponent<MoveComponent>();
    
    // 计算停止位置
    float3 targetPos = target.Position;
    float3 direction = math.normalize(targetPos - unit.Position);
    float distance = math.distance(unit.Position, targetPos);
    targetPos = unit.Position + direction * Math.Max(0, distance - stopDistance);
    
    // 设置目标（组件开始自主移动）
    move.SetTarget(targetPos);
    
    // 持续监控距离
    while (true)
    {
        if (!move.CanMove) return false;
        
        float currentDist = math.distance(unit.Position, target.Position);
        if (currentDist <= stopDistance + 0.5f)
        {
            // 到达攻击范围
            move.TargetPosition = null;  // 停止移动
            return true;
        }
        
        if (target.IsDisposed) return false;
        
        await TimerComponent.Instance.WaitAsync(100);
    }
}
```

### 寻找最近目标

```csharp
public static Unit FindNearestServerTarget(Unit unit, float searchRange)
{
    RoomUnitComponent unitComponent = unit.GetParent<RoomUnitComponent>();
    if (unitComponent == null) return null;

    Unit nearest = null;
    float minDistance = searchRange;

    foreach (Entity entity in unitComponent.Children.Values)
    {
        if (entity is not Unit other) continue;
        if (other.Type() != UnitType.Player) continue;

        StatusFlagsComponent otherState = other.GetComponent<StatusFlagsComponent>();
        if (otherState != null && otherState.IsDead) continue;

        float distance = math.distance(unit.Position, other.Position);
        if (distance < minDistance)
        {
            minDistance = distance;
            nearest = other;
        }
    }

    // 找到目标后，直接设置
    if (nearest != null)
    {
        MoveComponent move = unit.GetComponent<MoveComponent>();
        move.SetTarget(nearest.Position);
    }

    return nearest;
}
```

## ⚡ 性能优化要点

### 1. 自驱动优势

```csharp
// 旧设计：外部传递路径（每次都需要构建路径点）
List<float3> path = new List<float3> { p1, p2, p3 };
await move.MoveToAsync(path, speed);  // ❌ 耦合度高

// 新设计：只需设置目标（组件内部自动处理）
```


```csharp
// 组件内部只在需要时寻路
if (Targets.Count == 0 && TargetPosition != null)

```

## 🔍 源码关键部分

### SetTarget 方法

```csharp
public static bool SetTarget(this MoveComponent self, float3 target)
{
    // 距离检查
    if (math.distance(self.GetParent<Unit>().Position, target) < 0.1f)
    {
        self.TargetPosition = null;  // 已经到达
        return false;
    }

    // 只需设置目标，组件会自动处理其余逻辑
    self.TargetPosition = target;
    return true;
}
```

### MoveForward 自动寻路逻辑

```csharp
private static void MoveForward(this MoveComponent self, bool ret)
{
    if (!self.CanMove)
    {
        if (self.TargetPosition == null)
        {
            self.MoveFinish(ret);
        }
        return;  // 被锁定或无目标
    }

    Unit unit = self.GetParent<Unit>();
    
    // 自动寻路（仅首次或路径用完）
    if (self.Targets.Count == 0 && self.TargetPosition != null)
    {
        PathfindingManagerComponent pathfinding = unit.GetComponent<PathfindingManagerComponent>();
        if (pathfinding != null)
        {
            M2C_PathfindingResult result = M2C_PathfindingResult.Create();
            pathfinding.Find(unit.Position, self.TargetPosition.Value, result.Points);
            
            if (result.Points.Count >= 2)
            {
                foreach (float3 point in result.Points)
                {
                    self.Targets.Add(point);
                }
            }
            else
            {
                // 直接移动到目标
                self.Targets.Add(unit.Position);
                self.Targets.Add(self.TargetPosition.Value);
            }
        }
    }
    
    // ... 执行移动插值 ...
}
```

## 📊 设计对比

| 特性 | 旧设计（MoveToAsync） | 新设计（SetTarget） |
|------|----------------------|---------------------|
| **使用方式** | 传递路径点列表 | 只设置目标位置 |
| **寻路时机** | 外部调用前寻路 | 组件内部自动寻路 |
| **耦合度** | 高（外部需要构造路径） | 低（只关心目标） |
| **动态调整** | 需要取消+重新调用 | 更新 TargetPosition 即可 |
| **AI 集成** | 需要管理路径点 | 只需设置/清空目标 |
| **代码量** | 多（构造路径） | 少（一行代码） |

## 📝 总结

`MoveComponent` 采用**自驱动架构**，实现了真正的组件化设计：

**核心特性**：
- ✅ **目标驱动**：`SetTarget(target)` 一行代码完成移动
- ✅ **自动寻路**：组件内部调用 `PathfindingManagerComponent`
- ✅ **逻辑锁机制**（MoveForbiddenCount）支持 AI 控制
- ✅ **解耦设计**：外部不关心路径点细节
- ✅ **动态调整**：随时更新 TargetPosition

**最佳实践**：
- 使用 `SetTarget(target)` 设置目标
- AI 攻击时加锁并清空目标：`move.TargetPosition = null`
- `finally` 中解锁，组件自动恢复移动
- 大量怪物启用 `ServerTimeSlicingComponent`
