# 状态机循环依赖修复文档

## 问题描述

### 1. 客户端状态组件问题
客户端错误地添加了状态机组件（StateMachineComponent），违反了"客户端不需要状态组件"的设计原则。

### 2. 循环依赖错误 (ET0013)
编译时出现静态类函数引用的环形依赖错误：

```
MoveComponentSystem → StateMachineComponentSystem → IdleStateComponentSystem → MoveComponentSystem
```

具体依赖链：
- `MoveComponentSystem.MoveToAsync()` 调用 `StateMachineComponent.ChangeState()`
- `StateMachineComponentSystem.CheckCanTransitionTo()` 调用 `IdleStateComponent.CanTransitionTo()`
- `IdleStateComponentSystem.OnStateEnter()` 调用 `MoveComponent.Stop()`

## 解决方案

### 一、移除客户端状态组件

#### 1.1 删除客户端状态机系统
**文件**: `Unity/Assets/Scripts/Hotfix/Client/Demo/Main/Unit/StateMachineComponentSystem.cs`

**操作**: 完全删除该文件及其 .meta 文件

**原因**: 客户端不应该维护状态机，状态应该完全由服务器驱动

#### 1.2 修改客户端 UnitFactory
**文件**: `Unity/Assets/Scripts/Hotfix/Client/Demo/Main/Unit/UnitFactory.cs`

**修改前**:
```csharp
if (unitInfo.Type == (int)UnitType.Enemy)
{
    unit.AddComponent<StateMachineComponent>(); // 客户端状态机
}
```

**修改后**:
```csharp
// 客户端不需要状态机组件，状态由服务器驱动
// 表现层通过动画事件或其他方式处理
```

#### 1.3 修改状态变化消息处理器
**文件**: `Unity/Assets/Scripts/Hotfix/Client/Demo/Battle/Handler/M2C_MonsterStateChangeHandler.cs`

**修改前**:
```csharp

if (stateMachine != null)
{

```

**修改后**:
```csharp

// AnimatorComponent animator = unit.GetComponent<AnimatorComponent>();

```

### 二、解除服务端循环依赖

#### 2.1 移除 MoveComponentSystem 对 StateMachineComponent 的直接调用
**文件**: `Unity/Assets/Scripts/Hotfix/Server/Demo/Move/MoveComponentSystem.cs`

**修改**: 在 `MoveToAsync()` 方法中移除以下代码：

```csharp
// 删除：通知状态机切换到 Move 状态
Unit unit = self.GetParent<Unit>();
StateMachineComponent stateMachine = unit.GetComponent<StateMachineComponent>();
stateMachine?.ChangeState(MachineState.Move);

// 删除：移动完成，通知状态机切换到 Idle 状态
stateMachine?.ChangeState(MachineState.Idle);
```

**保留**: 仅保留事件发布逻辑：
```csharp
EventSystem.Instance.Publish(scene, new MoveStart() {Unit = self.GetParent<Unit>()});
// ... 移动逻辑 ...
EventSystem.Instance.Publish(scene, new MoveStop() {Unit = self.GetParent<Unit>()});
```

#### 2.2 移除 IdleStateComponentSystem 对 MoveComponent 的直接调用
**文件**: `Unity/Assets/Scripts/Hotfix/Server/Demo/Unit/MachineStatus/IdleStateComponentSystem.cs`

**修改前**:
```csharp
public static void OnStateEnter(this IdleStateComponent self, Unit unit)
{
    self.EnterTime = TimeInfo.Instance.ServerNow();
    Log.Debug($"[IdleState] Enter: UnitId={unit.Id}");
    
    // 停止移动
    MoveComponent move = unit.GetComponent<MoveComponent>();
    if (move != null && !move.IsArrived())
    {
        move.Stop(false);
    }
}
```

**修改后**:
```csharp
public static void OnStateEnter(this IdleStateComponent self, Unit unit)
{
    self.EnterTime = TimeInfo.Instance.ServerNow();
    Log.Debug($"[IdleState] Enter: UnitId={unit.Id}");
    
    // 不在这里直接停止移动，避免循环依赖
    // 移动停止应该由外部逻辑或AI控制
}
```

#### 2.3 创建事件驱动的状态更新机制

**新建文件 1**: `Unity/Assets/Scripts/Hotfix/Server/Demo/Unit/MoveStart_UpdateStateMachine.cs`

```csharp
namespace ET.Server
{
    [Event(SceneType.Map)]
    public class MoveStart_UpdateStateMachine : AEvent<Scene, MoveStart>
    {
        protected override async ETTask Run(Scene scene, MoveStart args)
        {
            Unit unit = args.Unit;
            StateMachineComponent stateMachine = unit.GetComponent<StateMachineComponent>();
            if (stateMachine != null)
            {
                stateMachine.ChangeState(MachineState.Move);
            }
            
            await ETTask.CompletedTask;
        }
    }
}
```

**新建文件 2**: `Unity/Assets/Scripts/Hotfix/Server/Demo/Unit/MoveStop_UpdateStateMachine.cs`

```csharp
namespace ET.Server
{
    [Event(SceneType.Map)]
    public class MoveStop_UpdateStateMachine : AEvent<Scene, MoveStop>
    {
        protected override async ETTask Run(Scene scene, MoveStop args)
        {
            Unit unit = args.Unit;
            StateMachineComponent stateMachine = unit.GetComponent<StateMachineComponent>();
            if (stateMachine != null)
            {
                stateMachine.ChangeState(MachineState.Idle);
            }
            
            await ETTask.CompletedTask;
        }
    }
}
```

## 架构改进

### 修改前的依赖关系（循环依赖）

```
┌─────────────────────┐
│ MoveComponentSystem │
└──────────┬──────────┘
           │ 直接调用 ChangeState()
           ▼
┌──────────────────────────┐
│ StateMachineComponentSys │
└──────────┬───────────────┘
           │ 调用 CanTransitionTo()
           ▼
┌──────────────────────────┐
│ IdleStateComponentSystem │
└──────────┬───────────────┘
           │ 调用 Stop()
           ▼
┌─────────────────────┐
│ MoveComponentSystem │ ← 形成循环！
└─────────────────────┘
```

### 修改后的依赖关系（单向依赖）

```
┌─────────────────────┐
│ MoveComponentSystem │
└──────────┬──────────┘
           │ 发布事件
           ▼
┌─────────────────────────┐
│ MoveStart/Stop Events   │
└──────────┬──────────────┘
           │ 事件订阅
           ▼
┌──────────────────────────────┐
│ MoveStart_UpdateStateMachine │
│ MoveStop_UpdateStateMachine  │
└──────────┬───────────────────┘
           │ 调用 ChangeState()
           ▼
┌──────────────────────────┐
│ StateMachineComponentSys │
└──────────┬───────────────┘
           │ 调用状态方法
           ▼
┌──────────────────────────┐
│ IdleStateComponentSystem │
│ MoveStateComponentSystem │ ← 只读取 MoveComponent 属性
└──────────────────────────┘
```

## 设计原则

### 1. 客户端-服务器分离
- **客户端**: 纯表现层，不维护游戏逻辑状态
- **服务器**: 权威状态管理，通过消息同步给客户端

### 2. 事件驱动架构
- 使用事件系统解耦模块间的直接依赖
- 发布者不需要知道订阅者的存在
- 便于扩展和维护

### 3. 单向依赖原则
- 避免循环依赖，保持依赖关系清晰
- 低层模块不应该依赖高层模块
- 使用事件或接口反转依赖

### 4. 职责分离
- **MoveComponent**: 负责移动逻辑和路径计算
- **StateMachineComponent**: 负责状态转换和状态管理
- **StateComponents**: 负责各状态的具体行为
- **Event Handlers**: 负责模块间的通信协调

## 注意事项

### 1. 状态转换时机
- 移动开始时，通过 `MoveStart` 事件触发状态转换为 `Move`
- 移动结束时，通过 `MoveStop` 事件触发状态转换为 `Idle`
- AI 或其他逻辑可以通过事件系统触发状态转换

### 2. 状态同步
- 服务器状态变化后，需要通过 `M2C_MonsterStateChange` 消息同步给客户端
- 客户端接收到消息后，直接更新表现层（动画、特效等）

### 3. 扩展性
如果需要添加新的状态转换触发点：
1. 发布相应的事件（如 `AttackStart`、`AttackStop`）
2. 创建对应的事件处理器更新状态机
3. 不要在业务逻辑中直接调用 `StateMachineComponent.ChangeState()`
