# 配置表体系说明

> 最后更新: 2026-04-13

## 1. 概述

项目使用 Luban 从 Excel 导出配置，生成 C# 代码 + 二进制数据。配置体系围绕战斗系统建立，核心关系链：

```
StageConfig → WaveConfig → SpawnConfig → MonsterUnitConfig → UnitCombatConfig → SkillConfig → BuffGroupConfig → BuffConfig
```

## 2. 配置表一览

### 2.1 战斗流程配置

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| StageConfig | 关卡配置，管理波次列表 | Id, Name, Type, WaveList, Reward |
| WaveConfig | 波次配置，管理刷怪批次 | Id, WaveNumber, Batches(list,BatchInfo), WaveInterval |
| SpawnConfig | 刷怪表，管理怪物类型和数量 | Id, PositionX, PositionZ, SpreadRange, Monsters(list,MonsterSpawnInfo) |

### 2.2 单位配置

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| MonsterUnitConfig | 怪物基础属性 | Id, Name, Hp, Attack, Defense, Speed, Type |
| UnitCombatConfig | 战斗行为配置 | Id, ConfigId, NormalAttackSkillId, AutoSkillIds, MoveSpeed, AutoCastNormalAttack |

### 2.3 技能配置

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| SkillConfig | 技能定义 | Id, Name, CastType(瞬发/投射物), CooldownMs, BuffGroupId, TargetingConfigId |
| SkillTargetingConfig | 目标选取 | Id, SelectType(锁定/最近/范围), CastRange, EdgeDistance, MaxTargetCount, TargetCampRelation |

### 2.4 效果配置

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| **BuffConfig** | 效果原子（最小执行单元） | Id, EffectType, FormulaType, BaseValue, RatioAtk, RatioDef, **Duration**, **TickInterval**, **MaxStack** |
| **BuffGroupConfig** | 效果组（多个效果原子组合） | Id, EffectIds(array), ExecMode |

### 2.5 其他配置

| 表名 | 用途 |
|------|------|
| BattleGlobalConfig | 全局战斗参数（如 SlotSpacing） |
| AIConfig | AI 行为树配置 |
| ResourceConfig | 资源定义 |
| UnitConfig | 玩家单位配置 |

## 3. 配置关系详解

### 3.1 UnitCombatConfig → SkillConfig

```
UnitCombatConfig (战斗行为)
  ├── NormalAttackSkillId → SkillConfig (普攻技能)
  └── AutoSkillIds[] → SkillConfig[] (AI自动技能列表)
```

### 3.2 SkillConfig → BuffGroupConfig → BuffConfig

```
SkillConfig (技能)
  └── BuffGroupId → BuffGroupConfig (效果组)
                        └── EffectIds[] → BuffConfig[] (效果原子列表)
                                            ├── [0] Damage (伤害)
                                            ├── [1] SlowDown (减速)
                                            └── [2] DOT (持续伤害)
```

**一个技能可以包含多个效果原子**，按顺序执行。

### 3.3 StageConfig → WaveConfig → SpawnConfig

```
StageConfig (关卡)
  └── WaveList[] → WaveConfig[] (波次列表)
                      └── Batches[] → BatchInfo(SpawnId, Delay)
                                        └── SpawnConfig (刷怪表)
                                              └── Monsters[] → MonsterSpawnInfo(MonsterId, Count)
                                                                 └── MonsterUnitConfig (怪物属性)
```

### 3.4 MonsterUnitConfig → UnitCombatConfig

```
MonsterUnitConfig (怪物基础属性: HP/Attack/Defense/Speed)
  └── ConfigId → UnitCombatConfig (战斗行为: 技能列表/移动速度/AI行为)
                    └── NormalAttackSkillId / AutoSkillIds → SkillConfig
```

## 4. BuffConfig 字段说明

BuffConfig 是效果的最小配置单元，所有战斗效果都通过此表定义。

| 字段 | 类型 | 说明 | 使用场景 |
|------|------|------|----------|
| Id | int | 唯一ID | 全局引用 |
| EffectType | int | 效果类型（见下表） | 决定执行逻辑分支 |
| FormulaType | int | 公式类型：1=攻防公式，2=固定值 | Damage/LifeSteal/DOT |
| BaseValue | float | 基础值 | 伤害值/治疗量/减速比例/护盾量/击退距离 |
| RatioAtk | float | 攻击系数 | 攻防公式中乘以攻击力 |
| RatioDef | float | 防御系数 | 攻防公式中乘以防御力 |
| MinValue | int | 最小值 | 伤害保底 |
| MaxValue | int | 最大值（0=无上限） | 伤害上限 |
| **Duration** | int | 持续时间(ms) | 持续效果专用，0=即时 |
| **TickInterval** | int | tick间隔(ms) | DOT专用 |
| **MaxStack** | int | 最大叠层数 | 可叠加的buff |
| CanCritical | bool | 是否可暴击 | 预留字段 |
| ApplyTargetFilter | string | 目标过滤 | 预留字段 |
| Desc | string | 描述 | 策略备注 |

### EffectType 枚举与 BaseValue 语义

| EffectType | BaseValue 含义 | Duration 含义 |
|------------|---------------|---------------|
| Damage | 附加基础伤害 | 不使用（即时） |
| Freeze | 不使用 | 冻结持续时间(ms) |
| Knockback | 击退距离(米) | 不使用（即时） |
| Heal | 治疗量 | 不使用（即时） |
| Stun | 不使用 | 眩晕持续时间(ms) |
| SlowDown | 减速百分比(0~1) | 减速持续时间(ms) |
| LifeSteal | 附加基础伤害 | 不使用（即时） |
| Shield | 护盾吸收量 | 护盾持续时间(ms) |
| AttackBuff | 增加攻击力值 | 增攻持续时间(ms) |
| DefenseBuff | 增加防御力值 | 增防持续时间(ms) |
| DOT | 附加基础伤害 | 持续伤害总时长(ms) |

## 5. 生成与使用

### 生成命令

```bash
# 服务端（C#代码 + 二进制数据）
./Tools/Luban/GenConfig_Server.sh

# 客户端（C#代码 + 二进制数据）
./Tools/Luban/GenConfig_Client.sh
```

### 生成文件位置

| 目标 | C# 代码 | 二进制数据 |
|------|---------|-----------|
| 服务端 | `Server/Model/Generate/Config/` | `Config/Generate/` |
| 客户端 | `Unity/Assets/GameScripts/HotFix/GameProto/Generate/Config/` | `Unity/Assets/AssetRaw/Configs/` |

### 代码中访问配置

```csharp
// 获取单个 BuffConfig
BuffConfig config = BuffConfigCategory.Instance.Get(buffId);

// 获取 BuffGroupConfig 中的效果列表
BuffGroupConfig group = skillConfig.BuffGroupIdConfig;
foreach (int effectId in group.EffectIds)
{
    BuffConfig effect = BuffConfigCategory.Instance.Get(effectId);
    // effect.EffectType, effect.BaseValue, effect.Duration ...
}
```

## 6. ID 分配约定

| 范围 | 用途 |
|------|------|
| 1001-1999 | StageConfig 关卡 |
| 1-999 | WaveConfig 波次 |
| 101-999 | SpawnConfig 刷怪表 |
| 2001-2099 | SkillConfig 技能 |
| 21001+ | SkillTargetingConfig 目标选取 |
| 3001-3999 | MonsterUnitConfig / UnitCombatConfig |
| 51001-511999 | BuffConfig 效果（按 EffectType 细分） |
| 61001+ | BuffGroupConfig 效果组 |

## 8. 新增配置的步骤

1. 在对应 Excel 文件中添加数据行
2. 运行 `GenConfig_Server.sh` 和 `GenConfig_Client.sh`
3. 检查生成的 C# 代码是否正确（`git diff`）
4. 在代码中引用新配置
5. 编译验证：`dotnet build Server/Server.sln`
