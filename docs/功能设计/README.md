## 🔧 开发指南

### 添加新功能模块
1. ✅ 明确功能需求和目标
2. ✅ 设计功能架构和流程
3. ✅ 基于框架层实现功能
4. ✅ 编写功能设计文档

### 功能模块设计原则
1. **单一职责**：每个功能模块专注于一个明确的功能
2. **松耦合**：功能模块之间尽量减少依赖
3. **可配置**：通过配置表控制功能参数
4. **可扩展**：预留扩展接口，便于后续迭代

### 文档编写要求
1. ✅ 功能概述和目标
2. ✅ 架构设计和流程图
3. ✅ 关键代码实现
4. ✅ 配置说明
5. ✅ 使用示例

## 🎮 功能模块依赖关系

```
功能设计层
    ↓ 依赖
框架设计层
    ↓ 依赖
ET 框架
```

### 示例：AI 系统依赖
- **依赖框架**：状态机框架、移动系统框架、战斗系统框架
- **依赖 ET**：Entity-Component-System、事件系统、定时器

### 示例：肉鸽游戏依赖
- **依赖框架**：战斗系统框架、状态机框架
- **依赖 ET**：Scene 管理、配置表系统
