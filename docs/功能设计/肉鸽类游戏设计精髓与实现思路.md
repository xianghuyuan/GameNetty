# 肉鸽类游戏设计精髓与实现思路

> **核心定位**：自走棋/弹幕肉鸽风格 —— 玩家和怪物均为自动行为，玩家通过 Build 构建影响战斗结果

## 1. Roguelike 核心设计哲学

### 1.1 核心三要素

```
永久死亡（Permadeath）
    │
    ├── 单局完整体验：每次游戏都是从零开始的完整冒险
    ├── 高风险决策：死亡代价使每个选择都有意义
    └── 成长外循环：永久解锁与 Meta 进度提供长期目标

程序生成（Procedural Generation）
    │
    ├── 波次随机：怪物种类、数量、刷新位置
    ├── 道具随机：掉落、升级选项、商店内容
    └── Build随机：技能/被动的随机组合

Build 构建（Build Crafting）—— 本游戏核心
    │
    ├── 技能组合：不同自动技能的协同效应
    ├── 被动叠加：多个被动效果的数值/效果堆叠
    └── 策略深度：升级时选择最优技能/被动组合
```

### 1.2 自走棋肉鸽的"爽点"循环

```
战斗（自动） → 通关波次 → 升级选技能 → Build 变强 → 战斗（自动）
      │                        │                    │
      └────────────────────────┴────────────────────┘
                   观赏性 + 策略决策
```

### 1.3 自动战斗的设计要点

```
混合操作模式（自走 + 可控）
    │
    ├── 玩家干预点
    │       │
    │       ├── 战斗前：Build 构建（技能/装备选择）
    │       ├── 波次间：升级奖励选择（3选1）
    │       └── 战斗中：可切换自动/手动模式
    │
    ├── 自动行为（AI驱动）
    │       │
    │       ├── 移动：始终自动（自动寻敌/走位/追击）
    │       ├── 目标选择：始终自动（最近/最弱等策略）
    │       └── 攻击/技能：可选自动（AI自动释放）
    │
    ├── 手动操作（玩家可控）
    │       │
    │       ├── 模式切换：随时切换自动/手动模式
    │       ├── 普攻指令：手动模式下点击/按键攻击
    │       └── 技能释放：手动模式下选择技能释放
    │
    ├── 怪物行为（完全自动）
    │       │
    │       ├── 怪物Unit：自动移动、自动攻击、自动释放技能
    │       └── AI驱动：与玩家自动模式使用相同AI框架
    │
    ├── 观赏性
    │       │
    │       ├── 技能特效：华丽的技能释放
    │       ├── 数字反馈：伤害飘字、连击提示
    │       └── 战斗节奏：怪物潮涌、技能轮转
    │
    └── 策略深度
            │
            ├── 模式选择：自动挂机 vs 手动操作
            ├── 技能协同：A技能触发B技能
            ├── 数值堆叠：同类效果叠加
            └── 风险收益：高风险高回报的选择
```

### 2.2 玩家决策流程（混合模式）

```
服务端 Tick（每帧/每500ms）
    │
    ├─→ 模式检查
    │       │
    │       └── 当前模式：Auto（自动）or Manual（手动）
    │
    ├─→ 目标选择（始终自动）
    │       │
    │       ├── 扫描范围内敌人
    │       ├── 选择策略：最近/最弱/最强/随机
    │       └── 锁定目标
    │
    ├─→ 移动决策（始终自动）
    │       │
    │       ├── 检查：是否有目标
    │       ├── 检查：距离是否足够攻击
    │       ├── 执行：向目标移动（寻路）
    │       └── 广播：M2C_Move
    │
    ├─→ 攻击决策（根据模式）
    │       │
    │       ├── 【自动模式】AI自动决策
    │       │       │
    │       │       ├── 检查：目标是否存在
    │       │       ├── 检查：距离是否足够
    │       │       ├── 检查：技能CD / 资源消耗
    │       │       ├── 选择：普攻 or 技能（优先级/CD）
    │       │       └── 执行攻击
    │       │
    │       └── 【手动模式】等待玩家指令
    │               │
    │               ├── C2M_Attack（普攻指令）
    │               │       ├── 服务端校验：目标/距离/CD
    │               │       └── 执行普攻
    │               │
    │               └── C2M_CastSkill（技能指令）
    │                       ├── 服务端校验：技能ID/CD/消耗
    │                       └── 执行技能
    │
    ├─→ 伤害计算（统一处理）
    │       │
    │       ├── 基础伤害 = ATK × 技能倍率
    │       ├── 暴击判定：CRIT_RATE → ×CRIT_DMG
    │       ├── 减伤计算：DEF / (DEF + K)
    │       ├── Buff 加成：增伤/减伤 Buff
    │       └── 广播：M2C_Damage / M2C_SkillEffect
    │
    └─→ 玩家其他输入
            │
            ├── C2M_SwitchControlMode（切换操作模式）
            ├── C2M_SelectReward（升级选择）
            ├── C2M_BuyItem（商店购买）
            └── C2M_StartBattle（开始战斗）
```

### 2.3 Build 构建系统

```
Build 层级
    │
    ├── 武器（主攻击方式）
    │       │
    │       ├── 近战类：剑/斧/锤 → 高伤/范围/击退
    │       ├── 远程类：弓/法杖 → 安全/持续/消耗
    │       └── 被动类：无需操作的自动武器
    │
    ├── 技能（主动释放）
    │       │
    │       ├── 伤害技能：AOE/单体/DOT
    │       ├── 控制技能：眩晕/减速/击退
    │       └── 辅助技能：护盾/回血/加速
    │
    ├── 被动（永久增益）
    │       │
    │       ├── 属性类：+10% ATK / +50 HP
    │       ├── 触发类：攻击时5%概率释放闪电
    │       └── 协同类：每持有1件火系装备+5%火伤
    │
    └── 遗物/圣物（特殊规则修改）
            │
            ├── 改变游戏规则：无法暴击但伤害+100%
            ├── 风险收益：HP上限-50%，伤害+80%
            └── 玩法拓展：解锁新的攻击模式
```

## 3. 怪物行为设计树

### 3.2 怪物类型设计

```
怪物分类
    │
    ├── 普通怪（Minion）
    │       │
    │       ├── 近战型：直线冲锋，接触伤害
    │       │       └── 行为：感知 → 追击 → 贴身攻击
    │       │
    │       ├── 远程型：保持距离，弹幕攻击
    │       │       └── 行为：感知 → 保持距离 → 发射弹幕
    │       │
    │       └── 自爆型：接近后自爆
    │               └── 行为：感知 → 冲锋 → 倒计时自爆
    │
    ├── 精英怪（Elite）
    │       │
    │       ├── 增强属性：HP ×3, ATK ×2
    │       ├── 特殊词缀：分裂/反射/闪现/嗜血
    │       └── 掉落增强：更高品质道具
    │
    └── Boss
            │
            ├── 多阶段：不同血量触发不同行为模式
            │       │
            │       ├── Phase 1 (100%-70%)：基础攻击模式
            │       ├── Phase 2 (70%-30%)：召唤小怪+新技能
            │       └── Phase 3 (30%-0%)：狂暴+必杀技
            │
            ├── 技能组合：
            │       │
            │       ├── 普攻：基础伤害
            │       ├── 技能1：范围AOE（前摇提示）
            │       ├── 技能2：弹幕地狱
            │       └── 必杀技：全屏攻击（需躲避点）
            │
            └── 特殊机制：
                    │
                    ├── 弱点暴露：特定时机可打出高伤害
                    ├── 护盾阶段：需要特定方式破盾
                    └── QTE事件：成功则打出高伤害
```

### 3.3 怪物 AI 决策流程（与玩家AI相同机制）

```
AI Tick（每 500ms）
    │
    ├── 前置检查
    │       │
    │       ├── 自身是否死亡？→ 停止AI
    │       ├── 是否被控制？→ 跳过决策
    │       └── 是否在技能释放中？→ 跳过决策
    │
    ├── 目标管理（自动）
    │       │
    │       ├── 当前目标有效？
    │       │       │
    │       │       ├── 是 → 继续攻击
    │       │       └── 否 → 寻找新目标
    │       │
    │       └── 寻找目标策略
    │               │
    │               ├── 最近距离
    │               ├── 最低血量
    │               ├── 仇恨值最高
    │               └── 随机选择
    │
    ├── 行为执行（自动）
    │       │
    │       ├── 距离 > 攻击范围
    │       │       └── 移动向目标（寻路）
    │       │
    │       ├── 距离 <= 攻击范围
    │       │       │
    │       │       ├── 普攻CD好了？→ 执行普攻
    │       │       ├── 技能CD好了？→ 释放技能
    │       │       └── 都在CD中？→ 等待/走位
    │       │
    │       └── Boss特殊逻辑
    │               │
    │               ├── 血量阈值检测 → 切换阶段
    │               └── 阶段技能序列 → 按顺序释放
    │
    └── 状态同步
            │
            └── 广播状态变化给房间内所有客户端
            
    注：玩家Unit和怪物Unit使用相同的AI决策机制
        区别仅在于配置参数（攻击范围、技能优先级等）
```

## 5. 奖励与成长系统

### 5.1 内循环成长（单局内）

```
单局成长路径
    │
    ├── 战斗奖励
    │       │
    │       ├── 击杀怪物 → 经验值 → 升级选技能
    │       ├── 完成波次 → 金币 → 商店购买
    │       └── 击杀Boss → 道具选择（三选一）
    │
    ├── 探索奖励
    │       │
    │       ├── 宝箱：随机道具/金币
    │       ├── 事件：风险收益选择
    │       └── 神坛：献祭换取强化
    │
    └── Build 强化节点
            │
            ├── 升级奖励：3选1技能/被动
            ├── 商店购买：道具/血量/技能升级
            └── 道具合成：低级→高级
```

```
永久成长系统
    │

    │       │
    │       ├── 新角色：不同初始属性/技能
    │       ├── 新武器：扩充武器池
    │       └── 新道具：扩充道具池
    │
    ├── 永久强化
    │       │
    │       ├── 属性升级：基础HP/ATK提升
    │       ├── 初始资源：起始金币/道具
    │       └── 特殊能力：复活次数/额外选择
    │
    └── 成就系统
            │
            ├── 挑战目标：首次通关/无伤通关
```
