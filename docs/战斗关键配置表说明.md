# 战斗关键配置表说明

本文档整理当前战斗系统中直接影响自动战斗、技能选择、施法判定与数值表现的关键配置表，重点说明字段语义、表间引用关系与排查顺序。

## 1. 文档范围

本文档基于当前生成代码整理，覆盖以下配置：

- `UnitCombatConfig`
- `AutoBattleStrategyConfig`
- `SkillConfig`
- `SkillTargetingConfig`

并结合以下代码说明使用方式：

- `Server/Hotfix/Demo/Battle/BattleSkillHelper.cs`

## 2. 配置关系总览

```text
UnitCombatConfig
 ├─ NormalAttackSkillId -> SkillConfig
 ├─ AutoSkillIds[] -> SkillConfig
 └─ AutoBattleStrategyId -> AutoBattleStrategyConfig

SkillConfig
 ├─ TargetingConfigId -> SkillTargetingConfig
 ├─ CastCheckConfigId -> SkillCastCheckConfig
 └─ EffectGroupId -> SkillEffectGroupConfig
```

其中本次文档重点覆盖与“战斗决策和同步链路”最直接相关的前四张表。

## 3. UnitCombatConfig

生成代码：`Server/Model/Generate/Config/UnitCombatConfig.cs`

### 3.1 作用

定义某个战斗单位在当前战斗框架下的基础战斗行为入口，包括：

- 普攻技能
- 自动技能列表
- 自动战斗策略
- 移动速度

`BattleSkillHelper` 中大量逻辑都先从 `UnitCombatConfig` 起跳。

### 3.2 关键字段

| 字段 | 含义 | 当前用途 |
|---|---|---|
| `ConfigId` | 关联战斗单位配置 ID | 用于按单位配置查到战斗行为定义 |
| `NormalAttackSkillId` | 默认普攻技能 ID | 普攻回退与攻击距离初始化 |
| `AutoSkillIds` | 自动战斗可选技能列表 | 自动战斗择优技能池 |
| `AutoBattleStrategyId` | 自动战斗策略配置 | 决定选技能 / 选目标规则 |
| `MoveSpeed` | 移动速度（米/秒） | 会同步写入 `NumericType.Speed` |
| `AutoCastNormalAttack` | 是否自动释放普攻 | 当前代码中字段已存在，实际逻辑需结合后续实现审视 |
| `DefaultTargetPolicy` | 默认目标策略 | 当前文档范围内未看到直接消费点 |
| `Desc` | 描述 | 配置备注 |

### 3.3 代码中的主要使用点

#### `ApplyNormalAttackConfig`

- 给 `BattleUnitCombatComponent` 写入：
  - `AttackCooldown`
  - `AttackRange`
- 如果 `MoveSpeed > 0`，会写入 `NumericType.Speed`

#### `GetAutoSkillIds`

- 先读取 `AutoSkillIds`
- 再确保 `NormalAttackSkillId` 也加入自动技能池
- 因此即便自动技能列表为空，只要配置了普攻，也仍有自动战斗回退能力

### 3.4 排查建议

出现以下问题时先查 `UnitCombatConfig`：

- 单位完全不会自动攻击
- 单位自动战斗只会普攻或没有技能可放
- 单位移动速度与预期不符

## 4. AutoBattleStrategyConfig

生成代码：`Server/Model/Generate/Config/AutoBattleStrategyConfig.cs`

### 4.1 作用

定义自动战斗在“技能选择、目标选择、目标保持、冷却中预走位”上的策略偏好。

### 4.2 关键字段

| 字段 | 含义 | 当前用途 |
|---|---|---|
| `Code` | 策略代码 | 策略标识 |
| `SkillSelectRule` | 技能选择规则 | `TryCompareSkillSelectionRule` |
| `TargetSelectRule` | 目标选择规则 | `TryCompareTargetSelectionRule` |
| `PreferBoss` | 是否优先 Boss | 提升 Boss 目标权重 |
| `PreferElite` | 是否优先精英 | 提升精英目标权重 |
| `PreferCurrentTarget` | 是否优先保持当前目标 | 减少频繁切目标 |
| `AllowPreMoveOnCooldown` | 是否允许技能冷却中预走位 | 决定 readyOnly=false 的二次尝试 |
| `TargetSwitchTolerance` | 切换目标容差距离 | 控制保持当前目标的容忍度 |
| `Desc` | 描述 | 配置备注 |

### 4.3 代码中的主要使用点

#### `TrySelectBestAutoSkillPlan`

- 先尝试仅选择“当前可立即释放”的技能。
- 如果没有可用技能，且 `AllowPreMoveOnCooldown=true`，允许为了冷却中的技能先走位。

#### `TryCompareCurrentTargetPreference`

- 如果配置要求优先保持当前目标，并且切换目标带来的收益没有明显超过容差，则继续保留当前目标。

#### `GetTargetTypeWeight`

- `PreferBoss` / `PreferElite` 会为目标权重加分，影响最终目标选择。

### 4.4 排查建议

出现以下问题时优先查 `AutoBattleStrategyConfig`：

- AI 总是频繁换目标
- AI 不优先打 Boss / 精英
- 技能还在冷却时单位不愿提前靠近目标
- 技能选择顺序与预期不一致

## 5. SkillConfig

生成代码：`Server/Model/Generate/Config/SkillConfig.cs`

### 5.1 作用

定义技能本体，是施法链路的核心配置入口，连接目标规则、施法校验和效果组。

### 5.2 关键字段

| 字段 | 含义 | 当前用途 |
|---|---|---|
| `Id` | 技能 ID | 全链路主索引 |
| `Code` | 技能代码名 | 配置识别 |
| `Name` | 技能名称 | 维护参考 |
| `SkillKind` | 技能类型 | 影响伤害类型展示 |
| `CastType` | 施法类型 | 预留给施法行为分类 |
| `TargetType` | 目标类型 | 技能目标语义 |
| `TargetingConfigId` | 目标配置引用 | 决定距离 / 目标筛选 |
| `CastCheckConfigId` | 施法校验配置引用 | 决定冷却 / 攻击限制 |
| `CooldownGroupId` | 冷却组 ID | 冷却体系字段 |
| `CooldownMs` | 冷却时间 | 直接参与冷却计算 |
| `Priority` | 优先级 | 自动战斗择优的重要依据 |
| `CanMoveCast` | 是否可移动施法 | 当前文档范围内未见直接消费点 |
| `NeedExplicitTarget` | 是否必须显式指定目标 | 影响 `SelectTargets` |
| `EffectGroupId` | 效果组引用 | 决定实际伤害 / 效果 |
| `IsEnabled` | 是否启用 | 技能总开关 |
| `Desc` | 描述 | 配置备注 |

### 5.3 代码中的主要使用点

#### `TryExecuteSkill`

- 技能是否存在、是否启用
- 读取目标配置 / 校验配置 / 效果组配置
- 写入当前 `AttackCooldown` / `AttackRange`
- 冷却校验与最终执行

#### `TryCompareSkillSelectionRule`

- 自动战斗在多个技能之间择优时，会比较：
  - `RequiredMoveDistance`
  - `Priority`

### 5.4 排查建议

出现以下问题时优先查 `SkillConfig`：

- 技能完全不可用
- 技能冷却时间异常
- 自动战斗选择了错误技能
- 技能打出来却不是预期伤害类型

## 6. SkillTargetingConfig

生成代码：`Server/Model/Generate/Config/SkillTargetingConfig.cs`

### 6.1 作用

定义技能如何找目标、能打多远、可命中多少个对象，以及筛选和排序规则。

### 6.2 关键字段

| 字段 | 含义 | 当前用途 |
|---|---|---|
| `SelectType` | 选目标方式 | 锁定目标 / 最近敌人 / 范围敌人 |
| `MaxTargetCount` | 最大目标数 | 命中数量上限 |
| `CastRange` | 基础施法距离 | 是否可释放的基础距离 |
| `EdgeDistance` | 边缘补偿距离 | 放宽命中边界 |
| `UseCollisionRadius` | 是否使用碰撞半径 | 当前代码会参与允许距离计算，但半径函数暂返回 0 |
| `SearchRadius` | 搜索半径 | 预留给范围搜索语义 |
| `SectorAngle` | 扇形角度 | 预留给扇形技能 |
| `TargetCampRelation` | 目标阵营关系 | 敌方 / 友方 / 自身 / 任意 |
| `RequireAlive` | 是否要求目标存活 | 过滤死亡目标 |
| `SortRule` | 排序规则 | 最近 / 最低血量 |
| `Desc` | 描述 | 配置备注 |

### 6.3 代码中的主要使用点

#### `IsInSkillRange`

- 允许距离 = `CastRange + EdgeDistance (+碰撞半径)`

#### `GetDesiredCastDistance`

- 在允许施法距离基础上，再减去一定缓冲，得到 AI 追击时希望停下来的位置

#### `SelectTargets`

- 根据 `SelectType` 选择：
  - 显式锁定目标
  - 最近敌人
  - 范围内全部敌人
- 根据 `SortRule` 排序
- 根据 `MaxTargetCount` 截断

### 6.4 当前实现注意事项

- `UseCollisionRadius` 虽然已接入距离计算，但 `GetCollisionRadius()` 当前返回 `0f`，说明碰撞半径逻辑还未真正落地。
- `SearchRadius`、`SectorAngle` 字段已在表结构中存在，但当前文档范围内未见完整扇形/圆形筛选实现。

这类字段应视为“结构已预留，但需结合实际代码确认是否生效”。

### 6.5 排查建议

出现以下问题时优先查 `SkillTargetingConfig`：

- 明明贴脸却提示目标超距
- 技能应该群攻却只打到一个目标
- 自动战斗总站得太远或太近
- 技能总是打错阵营

## 7. 推荐排查顺序

### 7.1 AI 不会主动攻击

1. `UnitCombatConfig` 是否存在
2. `NormalAttackSkillId` / `AutoSkillIds` 是否有效
3. `AutoBattleStrategyId` 是否能解析到策略
4. 对应 `SkillConfig` 是否启用

### 7.2 技能一直提示超距

1. `SkillTargetingConfig.CastRange`
2. `SkillTargetingConfig.EdgeDistance`
3. `BattleSkillHelper.GetDesiredCastDistance`
4. `BattleMoveComponent.FollowTarget` 是否把单位带到预期站位

### 7.3 AI 总换目标

1. `AutoBattleStrategyConfig.PreferCurrentTarget`
2. `TargetSwitchTolerance`
3. `TargetSelectRule`
4. 当前技能优先级是否驱动了目标切换

### 7.4 明明有技能却一直只普攻

1. `UnitCombatConfig.AutoSkillIds` 是否包含该技能
2. `SkillConfig.IsEnabled` 是否为 true
3. `SkillConfig.Priority` 是否太低
4. `SkillCastCheckConfig` 是否阻止释放
5. `SkillTargetingConfig` 是否导致技能难以命中

## 8. 维护建议

- 新加单位战斗行为时，优先补齐 `UnitCombatConfig`，不要只配角色基础属性。
- 新加自动战斗风格时，尽量通过 `AutoBattleStrategyConfig` 扩展，而不要写死在代码分支里。
- 新加技能时，要同时检查 `SkillConfig`、`SkillTargetingConfig`、`SkillCastCheckConfig`、`SkillEffectGroupConfig` 是否形成完整闭环。

## 9. 一句话总结

当前战斗配置的入口是 `UnitCombatConfig`，自动战斗偏好由 `AutoBattleStrategyConfig` 驱动，技能本体由 `SkillConfig` 定义，而“怎么找目标、能打多远、打多少个”则由 `SkillTargetingConfig` 决定。
