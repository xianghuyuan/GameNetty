## 📂 文档结构说明

### 框架设计 vs 功能设计

#### 🏗️ 框架设计
包含**底层架构、核心系统和通用组件**，这些是项目的基础设施：
- **战斗系统框架**：BattleRoom 架构、状态同步机制
- **状态机框架**：通用状态机系统
- **移动系统框架**：MoveComponent 移动组件
- **网络框架**：登录、网络架构、错误码
- **ET 框架**：ET 框架学习文档

**特点**：
- 通用性强，可复用
- 影响范围大，修改需谨慎
- 需要深入理解 ET 框架

#### 🎮 功能设计
包含**具体游戏玩法和业务逻辑**，这些是基于框架实现的功能：
- **AI 系统**：怪物 AI、割草游戏 AI
- **游戏玩法**：肉鸽游戏、关卡设计
- **视觉效果**：视差滚动

**特点**：
- 业务相关，针对性强
- 基于框架层实现
- 可独立开发和测试

## 📝 文档编写规范

### 命名规范
- 使用中文命名，语义清晰
- 特殊术语保留英文（如 ET、BattleRoom、AI）
- 格式：`功能模块名称.md` 或 `系统名称设计文档.md`

### 内容规范
- 包含文档版本号和更新日期
- 包含作者信息
- 代码示例使用 markdown 代码块
- 重要概念使用加粗或列表突出
- 包含目录结构（超过 3 个章节）

### 更新规范
- 重大更新需更新版本号
- 在文档末尾记录更新内容
- 同步更新本 README.md 的更新记录
