# 战斗进入过渡系统设计文档

## 1. 系统概述

本文档定义了玩家从主界面进入战斗、战斗进行中、以及战斗结束返回主界面的完整过渡流程。

### 1.1 设计原则

- **不切换 Unity 场景**：通过坐标隔离实现主世界和战斗区域的分离
- **华丽的视觉过渡**：使用 Spine 动画特效提供沉浸式的战斗开始体验
- **平滑的 UI 切换**：主界面 UI 动画隐退，战斗 UI 动画显现
- **固定战斗区域**：战斗在远离主世界的固定坐标区域进行
- **清晰的状态管理**：通过事件系统解耦各模块

### 1.2 技术方案总览

| 方面 | 方案 |
|------|------|
| 场景管理 | 不切换场景，固定战斗区域坐标隔离 |
| 过渡表现 | Spine 华丽动画特效 + Fade 效果 |
| UI 切换 | 主界面 UI 动画隐退 → 战斗 UI 动画显现 |
| 相机管理 | 单相机，黑屏期间瞬移到战斗区域 |

## 2. 坐标隔离方案

### 2.1 区域划分

```
主世界区域:
  中心坐标: (0, 0, 0)
  活动范围: ±1000

战斗区域:
  中心坐标: (10000, 0, 0)
  活动范围: ±50（固定战斗场地）

预留区域:
  副本区域: (20000, 0, 0)
  Boss区域: (30000, 0, 0)
```

### 2.2 坐标配置

```csharp
public static class BattleAreaConfig
{
    // 战斗区域中心
    public static readonly Vector3 BattleAreaCenter = new Vector3(10000, 0, 0);
    
    // 玩家出生点（战斗区域内相对坐标）
    public static readonly Vector3 PlayerSpawnOffset = new Vector3(0, 0, -5);
    
    // 怪物生成区域中心（战斗区域内相对坐标）
    public static readonly Vector3 MonsterSpawnOffset = new Vector3(0, 0, 10);
    
    // 战斗相机位置
    public static readonly Vector3 BattleCameraPos = new Vector3(10000, 8, -12);
    public static readonly Vector3 BattleCameraRotation = new Vector3(30, 0, 0);
}
```

### 2.3 坐标转换

服务器下发的是相对坐标（以战斗区域中心为原点），客户端需要加上偏移：

```csharp
Vector3 worldPos = serverRelativePos + BattleAreaConfig.BattleAreaCenter;
Vector3 serverPos = worldPos - BattleAreaConfig.BattleAreaCenter;
```

## 3. 状态机定义

```csharp
public enum BattleTransitionState
{
    None,                // 非战斗状态（主界面）
    RequestingBattle,    // 请求战斗中（等待服务器响应）
    FadingOut,           // 主界面淡出 + 屏幕渐黑
    PlayingTransition,   // 播放 Spine 战斗开始动画
    FadingIn,            // 战斗UI淡入 + 屏幕渐亮
    InBattle,            // 战斗进行中
    ShowingResult,       // 显示结算界面
    ReturningToMain,     // 返回主界面过渡中
}
```

## 4. 进入战斗流程

```
玩家点击"开始战斗"
    │
    ▼
Phase 1: 请求战斗 (RequestingBattle, ~0.1s + 网络延迟)
    │ - 发送 C2M_StartBattle 到服务器
    │ - 等待 M2C_StartBattle 响应，获得 battleId
    │ - 创建 Battle 实例
    ▼
Phase 2: 淡出主界面 (FadingOut, 0.3s)
    │ - 主界面 UI 播放隐退动画（缩小+淡出）
    │ - 屏幕 Fade Out（渐黑）
    │ - 黑屏后：关闭所有主界面 UI
    │ - 黑屏后：相机瞬移到战斗区域坐标
    │ - 黑屏后：切换战斗 BGM
    ▼
Phase 3: 播放过渡动画 (PlayingTransition, 1.0~1.5s)
    │ - 播放华丽的 Spine 战斗开始特效
    │ - 显示"战斗开始！"文字动画
    │ - 播放战斗开始音效
    ▼
Phase 4: 淡入战斗界面 (FadingIn, 0.5s)
    │ - 显示战斗 UI（BattleMainUI）
    │ - 战斗 UI 播放入场动画
    │ - 屏幕 Fade In（渐亮）
    ▼
Phase 5: 等待第一波 (服务器控制)
    │ - 收到 M2C_WaveStart
    │ - 显示波次信息："第 1/5 波"
    ▼
战斗进行中 (InBattle)
```

总进入时长：约 1.8~2.3s（不含网络延迟）

---

## 5. 退出战斗流程

```
收到 M2C_BattleEnd
    │
    ▼
Phase 6: 战斗结束 (BattleEnding)
    │ - 播放胜利/失败 Spine 动画
    │ - 播放对应音效
    ▼
Phase 7: 显示结算 (ShowingResult, 用户控制)
    │ - 弹出结算界面（BattleResultUI）
    │ - 显示：战斗结果、奖励、伤害统计
    │ - 等待玩家点击"确认"
    ▼
Phase 8: 返回主界面 (ReturningToMain, 0.8s)
    │ - 屏幕 Fade Out（渐黑，0.3s）
    │ - 关闭战斗 UI 和结算界面
    │ - 清理 Battle 实例和战斗单位
    │ - 相机瞬移回主世界坐标
    │ - 切换回主世界 BGM
    │ - 恢复主界面 UI
    │ - 屏幕 Fade In（渐亮，0.3s）
    ▼
主界面 (None)
```

总退出时长：约 0.8s（不含结算界面停留时间）

## 6. 事件系统设计

### 6.1 新增事件

在 `BattleEventType.cs` 中添加：

```csharp
// 战斗过渡开始（进入战斗，主界面开始隐退）
public struct BattleTransitionStart { public Battle Battle; }

// 战斗过渡完成（已进入战斗，战斗UI已显示）
public struct BattleTransitionComplete { public Battle Battle; }

// 战斗退出开始（开始返回主界面）
public struct BattleExitStart { public Battle Battle; }

// 战斗退出完成（已返回主界面）
public struct BattleExitComplete { public Battle Battle; }
```

### 6.2 事件流转

```
进入战斗:
  BattleTransitionStart → 主界面UI隐退、Fade Out、相机切换
      → BattleTransitionComplete → 战斗UI显示、Fade In

退出战斗:
  BattleExitStart → Fade Out、清理战斗、相机切换
      → BattleExitComplete → 主界面UI恢复、Fade In
```

## 7. UI 管理方案

### 7.1 UI 切换策略

进入战斗时：
1. 主界面 UI 播放隐退动画（缩小+淡出，0.3s）
2. 动画结束后关闭 UILayer.UI 层所有窗口
3. 显示战斗 UI（BattleMainUI），播放入场动画

退出战斗时：
1. 关闭 UILayer.Battle 层所有窗口
2. 恢复主界面 UI，播放入场动画（放大+淡入）

### 7.2 需要的 UI 列表

| UI 名称 | 层级 | 用途 | 状态 |
|---------|------|------|------|
| BattleMainUI | Battle | 战斗HUD（技能、血条等） | 已存在，需完善 |
| WaveInfoUI | Battle | 波次信息显示 | 待创建 |

| FadeUI | Transition | 黑屏渐变遮罩 | 待创建 |
| BattleTransitionUI | Transition | Spine战斗开始动画容器 | 待创建 |

## 8. 相机管理方案

### 8.1 切换策略

使用单相机，在黑屏期间瞬移：

```csharp
// 进入战斗：保存当前位置 → 瞬移到战斗区域
savedPos = camera.transform.position;
savedRot = camera.transform.rotation;
camera.transform.position = BattleAreaConfig.BattleCameraPos;
camera.transform.eulerAngles = BattleAreaConfig.BattleCameraRotation;

// 退出战斗：瞬移回保存的位置
camera.transform.position = savedPos;
camera.transform.rotation = savedRot;
```

### 8.2 时机

相机切换发生在 Phase 2（FadingOut）屏幕完全黑屏之后，玩家看不到切换过程。

## 9. Spine 动画特效方案

### 9.1 战斗开始动画

资源：`battle_start.skel`
播放位置：屏幕中央（UI层）
播放时机：Phase 3（PlayingTransition）

```
时间轴:
  0.0s ~ 0.3s  光效从中心扩散
  0.3s ~ 0.6s  "战斗开始！"文字弹出（缩放+抖动）
  0.6s ~ 1.0s  华丽粒子特效环绕
  1.0s ~ 1.2s  整体淡出
```

### 9.2 波次开始提示

资源：`wave_start.skel`
播放位置：屏幕上方
播放时机：收到 M2C_WaveStart 时

```
时间轴:
  0.0s ~ 0.2s  横幅从上方滑入
  0.2s ~ 0.5s  显示"第 X 波"文字 + 怪物数量
  0.5s ~ 0.8s  停留展示
  0.8s ~ 1.0s  横幅滑出
```

### 9.3 战斗结果动画

胜利资源：`battle_victory.skel`
失败资源：`battle_defeat.skel`
播放时机：收到 M2C_BattleEnd 时

```
胜利:
  金色光效爆发 → "胜利！"文字 → 星星粒子散落

失败:
  灰色碎裂效果 → "失败"文字淡入 → 画面轻微震动
```
