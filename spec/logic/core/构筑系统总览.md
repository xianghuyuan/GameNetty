# 构筑系统总览

> 最后更新: 2026-06-22

## 1. 核心定位

GameNetty 的核心玩法应统一理解为：玩家通过发射器、效果包和 Buff 原子组合，构筑一套局内战斗流派，再用怪潮和 Boss 验证构筑强度。

割草、波次和 Boss 是验证环境；真正驱动可玩深度的是构筑。

## 2. 术语边界

| 概念 | 代码对象 | 职责 |
|---|---|---|
| 角色 | `Unit` / `BattleUnit` | 谁在战斗，持有属性和运行时状态 |
| 构筑 | `BuildRuntime` | 进入战斗后解析出的可执行攻击组合 |
| 发射器 | `EmitterConfig` / `EmitterRuntime` | 攻击形态、CD、射程、投射物、白值伤害和槽位 |
| 效果包 | `EmitterEffectPackConfig` | 玩家装到发射器槽位上的构筑单元 |
| 发射器效果 | `EmitterEffectConfig` | 修改发射器属性，或命中时追加 BuffGroup |
| 效果组 | `BuffGroupConfig` | 一次命中携带的一组 Buff |
| 效果原子 | `BuffConfig` | 最终执行的最小战斗效果，如伤害、减速、DOT、吸血 |

玩家可见的“技能”不是单独的数据层。它等价于：

```text
技能 = 发射器 + 已装配的效果包 + 展开后的 BuffGroup
```

## 3. 配置引用关系

```text
UnitCombatConfig
  -> NormalAttackEmitterId
  -> AutoEmitterIds[]

EmitterConfig
  -> TargetingConfigId
  -> UpgradeConfigId
  -> BuffSlotCount

VehicleData
  -> VehicleConfigId  // EmitterConfig.Id
  -> SlottedEffectPackIds[]

EmitterEffectPackConfig
  -> EffectIds[]

EmitterEffectConfig
  -> EffectKind = 1 修改发射器属性
  -> EffectKind = 2 追加 BuffGroup
  -> BuffGroupId

BuffGroupConfig
  -> BuffIds[]

BuffConfig
  -> EffectType / FormulaType / Duration / TickInterval / MaxStack
```

## 4. 运行时链路

进入战斗或装备变化时，先把玩家当前载具解析成构筑运行时：

```text
VehicleComponent
  -> BuildRuntimeResolver
  -> BuildRuntime
      -> EmitterRuntime[]
```

战斗系统只消费 `EmitterRuntime` 转换后的 `BattleAttackRuntime`，不直接关心载具槽位和效果包如何展开。

```text
BattleAttackComponent
  -> BattleAttackRuntime
  -> 命中
  -> BuffGroupConfig
  -> BuffConfig
  -> BuffComponent / 即时效果执行
```

## 5. 模块边界

| 模块 | 职责 |
|---|---|
| `Module/Build` | 将玩家构筑解析成战斗运行时，不负责命中和伤害 |
| `Module/Vehicle` | 管理载具、槽位和效果包装配数据 |
| `Module/Battle` | 执行攻击、索敌、CD、命中和表现事件 |
| `Module/Buff` 或 Battle 内 Buff 相关类 | 执行 Buff 生命周期、tick、过期和属性还原 |
| `Module/Config` | 提供配置表读取入口 |

## 6. 新增内容的规则

新增一个发射器：

1. 在 `EmitterConfig.xlsx` 配攻击形态、CD、射程、投射物、白值和槽位。
2. 如有升级，在 `EmitterUpgradeConfig.xlsx` 配等级成长。
3. 不在发射器里写死具体 Buff 流派，流派通过效果包进入。

新增一个效果包：

1. 在 `EmitterEffectConfig.xlsx` 配一个或多个效果。
2. 属性修改类效果用 `EffectKind=1`。
3. 命中追加效果用 `EffectKind=2` 并引用 `BuffGroupConfig.Id`。
4. 在 `EmitterEffectPackConfig.xlsx` 把这些效果组合成一个可装配包。

新增一个 Buff 效果：

1. 如果是现有 `EffectType`，只新增 `BuffConfig` 行。
2. 如果是新行为，先扩展 `EffectType` 和执行器，再新增 `BuffConfig`。
3. 多个 Buff 需要一起命中时，用 `BuffGroupConfig` 组合。

## 7. 命名约定

代码层优先使用 `Emitter`、`EffectPack`、`BuffGroup`、`Buff`。

历史代码里的 `SkillId`、`TryExecuteSkill`、`BattleSkillHelper` 可逐步重命名为 `EmitterId`、`TryExecuteEmitter`、`BattleEmitterHelper`。重命名应分批做，避免一次性影响协议和服务端同步。
