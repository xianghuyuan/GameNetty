# 玩家动线与局外养成路线

> 最后更新: 2026-06-30

## 1. 核心路线

GameNetty 的主流程确定为：

```text
大厅看目标
  -> 章节选关
  -> 战前构筑
  -> 战斗验证
  -> 结算奖励
  -> 局外养成
  -> 构筑调整
  -> 再挑战
```

这条路线的核心定位是：玩家通过局外养成和构筑调整变强，再用章节波次和 Boss 战验证构筑强度。

章节闯关负责给玩家明确目标，局外养成负责给玩家失败后的改善路径，构筑系统负责提供长期可玩深度。

## 2. 玩家动线

### 2.1 大厅

大厅是玩家回流点，默认展示下一步最重要的行动：

- 当前章节进度。
- 下一关入口。
- 推荐战力或主要挑战点。
- 可强化提示，如发射器可升级、效果包可升级、槽位可解锁。

早期大厅不要放太多入口。第一版只保留主线章节、构筑/养成入口和必要的设置/返回功能。

### 2.2 章节选关

章节地图承担短期目标：

- 普通节点用于刷基础材料和推进进度。
- 精英节点用于验证阶段性强度。
- Boss 节点作为章节终点，产出关键解锁材料或新系统开放条件。

关卡失败后，玩家应回到结算或大厅，并看到明确的强化建议，而不是直接停留在失败状态。

### 2.3 战前构筑

战前构筑是进入战斗前的最后确认点：

- 选择当前启用的发射器。
- 给发射器装配效果包。
- 查看已解锁槽位。
- 查看当前构筑的主要标签或倾向，如范围、控制、持续伤害、生存。

第一版不需要做复杂构筑评分，但必须让玩家知道当前装备了什么、还能替换什么。

### 2.4 战斗验证

战斗用于验证玩家构筑：

- 小怪潮验证清怪效率和范围覆盖。
- 精英怪验证单体输出、生存和控制。
- Boss 验证持续输出、走位容错和核心构筑强度。

战斗 HUD 继续以生存状态、Boss 状态、波次进度和发射器状态为主，不在战斗中展开完整养成面板。

### 2.5 结算

结算负责把战斗反馈转化为下一步行动：

- 展示通关/失败结果。
- 展示获得的金币、发射器材料、效果包材料、槽位解锁材料。
- 展示可立即强化项。
- 提供“继续下一关”“去强化”“再挑战”三个主行动。

失败结算应强调“差一点”和“可改善项”，例如生命不足、输出不足、Boss 时间过长、清怪效率不足。

### 2.6 局外养成

局外养成是每次战斗后的主要回流点。第一版只做四条主线：

1. 角色等级。
2. 发射器升级。
3. 效果包升级。
4. 槽位解锁。

这四条养成线都必须能反向影响下一场战斗，避免出现只增加面板但玩家无感的成长。

## 3. 可养成内容优先级

### 3.1 第一版必须支持

| 养成项 | 作用 | 产出来源 |
|---|---|---|
| 角色等级 | 提升生命、攻击、防御等基础属性 | 普通关卡、首通奖励 |
| 发射器升级 | 提升伤害、CD、射程、投射物、范围等核心手感 | 章节材料、发射器碎片 |
| 效果包升级 | 提升 Buff 强度、触发率、持续时间或附加效果 | 指定章节掉落、Boss 掉落 |
| 槽位解锁 | 增加发射器槽、效果包槽或 Buff 挂载空间 | 章节进度、Boss 首通 |

### 3.2 后续扩展

后续可以扩展但不进入第一版主循环：

- 被动天赋树。
- 装备和词条。
- 多角色/英雄。
- 收藏、图鉴和羁绊。
- 活动副本、材料本、Boss 专项本。

这些系统必须在主循环稳定后再加入，否则会稀释“发射器 + 效果包 + 战斗验证”的核心体验。

## 4. 解锁节奏

建议第一章按以下节奏开放：

| 节点 | 开放内容 | 目的 |
|---|---|---|
| 1-1 | 基础战斗、角色升级 | 让玩家理解打关和变强 |
| 1-2 | 发射器升级 | 让玩家感知技能手感变化 |
| 1-3 | 效果包装配 | 引入构筑差异 |
| 1-4 | 效果包升级 | 让掉落材料形成目标 |
| 1-5 Boss | 解锁新槽位或新发射器 | 形成章节终点奖励 |

第二章开始再逐步引入不同关卡掉落、精英节点和针对性构筑需求。

## 5. 服务端底层逻辑

服务端需要负责以下权威数据：

- 玩家章节进度。
- 玩家资源和材料。
- 角色等级与基础属性。
- 发射器拥有、等级和升级消耗。
- 效果包拥有、等级和升级消耗。
- 槽位解锁状态。
- 战斗结算奖励与首通奖励。

推荐服务端按持久 `Unit` 维度保存局外养成数据。进入战斗时，由局外数据解析出本局 `BuildRuntime`，再同步到 `BattleRoom` 中的 `BattleUnit`。

战斗结束后，服务端根据关卡结果发放奖励，并更新章节进度、首通状态和材料数量。客户端只展示结果，不直接决定奖励。

## 6. 客户端底层逻辑

客户端需要提供以下模块入口：

- 大厅目标入口：展示下一关和可强化项。
- 章节地图入口：展示章节节点和锁定状态。
- 战前构筑入口：展示发射器、效果包和槽位。
- 局外养成入口：处理角色、发射器、效果包和槽位升级展示。
- 结算入口：展示奖励，并跳转到继续挑战、强化或再挑战。

客户端表现层通过服务端返回的玩家养成数据刷新 UI。构筑调整完成后，客户端保存选择并在进入战斗时提交给服务端校验。

## 7. 第一版验收标准

- 玩家登录后能看到下一关目标。
- 玩家能从大厅进入章节关卡。
- 玩家能在战前看到当前发射器和效果包。
- 战斗结束后能看到奖励和下一步按钮。
- 玩家能使用奖励升级至少一个发射器或效果包。
- 升级后再次进入战斗，伤害、CD、射程或效果强度至少有一项可感知变化。
- 第一章 Boss 首通后能解锁一个新槽位或新发射器。
