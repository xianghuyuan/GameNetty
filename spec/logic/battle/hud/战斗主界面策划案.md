# 战斗主界面策划案

## 1. 目标

战斗主界面用于承载玩家在横版割草战斗中的核心状态、战斗进度和即时反馈。界面应服务战斗判断，而不是覆盖舞台表现。

本界面需要解决：

- 玩家能快速看到自身生存状态。
- Boss 或关键敌人出现时，玩家能明确看到其血量状态。
- 玩家能知道当前战斗波次或关卡推进状态。
- 玩家能看到发射器/自动战斗相关状态。
- 战斗命中后有即时伤害反馈。
- 尽量减少对战斗区的遮挡。

## 2. 设计依据

当前没有独立旧版战斗主界面策划案。本文件基于以下资料整理并作为后续实现的主界面功能来源：

- `docs/gameplay.md`
- `docs/battle_flow.md`
- `spec/art/横版战斗背景视差设计.md`
- `spec/art/横版舞台资源接入规范.md`
- 当前 `BattleMainWindow` / `UIDamageWindow` / `UIDamageWidget` 实现

后续如效果图或交互规则调整，应同步更新本文档。

## 3. 界面原则

- HUD 是舞台前沿的小型道具系统，不是覆盖战斗画面的整屏面板。
- 默认态保持轻量，不长期占用过高屏幕比例。
- 右侧主要战斗区优先留给怪群、Boss、发射器弹道/预警。
- 信息优先级：生存状态 > Boss/关键目标 > 波次进度 > 发射器状态 > 装饰。
- 静态装饰与代码控制节点必须分离。
- UI 小图进入 `UIRaw/Atlas/Battle/battle_main/`，大图或不进图集资源进入 `UIRaw/Raw/Battle/`。
- 不使用 `Generated/` 目录作为正式资源目录。

## 4. 功能区

### 4.1 玩家状态区

用途：

- 展示玩家当前生命状态。
- 可选展示攻击、攻击范围或战斗力等调试/扩展信息。

本期要求：

- 常驻显示玩家血量。
- 血量采用血条 + 数字文本。
- 血条填充由代码控制。
- 血条背景、边框、图标为静态装饰。

默认状态：

- 玩家进入战斗后显示。
- 玩家血量变化时立即刷新。
- 战斗结束时清理。

### 4.2 Boss 状态区

用途：

- Boss 或关键敌人出现时展示其名称和血量。
- 让玩家明确 Boss 战阶段中的主要目标状态。

本期要求：

- 默认隐藏。
- Boss 出现后显示。
- Boss 死亡或战斗结束后隐藏/清理。
- Boss 血量变化必须以服务端确认数据为准。

暂不确认：

- 是否支持多管血。
- 是否支持阶段标记。
- 是否展示 Boss 蓄力状态。

### 4.3 波次信息区

用途：

- 展示当前波次进度，让玩家知道战斗推进情况。

本期要求：

- 显示当前波次 / 总波次。
- 可选显示当前波剩余怪物数。
- 响应 `M2C_WaveStart` 和 `M2C_WaveComplete`。

暂不确认：

- 是否需要进度条。
- 是否需要 Boss 节点或关卡节点。

### 4.4 发射器栏

用途：

- 展示玩家当前拥有/启用的发射器，以及每个发射器的自动攻击、冷却和可用状态。
- 发射器就是战斗中的主要技能表达；Buff 不作为独立技能，而是挂载在发射器上改变其表现或数值。

本期定位：

- 先保留发射器栏 prefab 结构和资源接入规则。
- 是否实现完整发射器 CD、图标、等级、锁定态，需要后续发射器系统界面需求确认。
- Buff 面板只负责选择和挂载到目标发射器，不在主 HUD 上形成另一套并列战斗槽。

建议：

- 发射器槽作为独立 `UIWidget`，不要在 `BattleMainWindow` 内重复绑定多个槽位字段。
- 发射器槽内可预留 1~3 个小型 Buff 挂载标记，用于表达该发射器当前已挂载的 Buff。
- Buff 挂载区必须依附当前选中的发射器槽，例如使用同一底板、选中高亮、连接线或槽内展开，不要做成与发射器栏无关联的独立面板。
- Buff 挂载操作区定位为轻量操作弹窗/抽屉：默认隐藏，只在点击或长按某个发射器槽后展开，用于挂载、替换、卸载或新增 Buff。
- 常驻 HUD 只显示发射器槽和槽内 Buff 小标记，不常驻显示完整 Buff 操作弹窗。

### 4.5 伤害飘字

用途：

- 展示战斗命中反馈。

当前状态：

- 已有 `UIDamageWindow` 和 `UIDamageWidget`。
- 通过 `BattleUnitDamaged` 事件显示伤害。

本期要求：

- 保持独立窗口，不塞进 `BattleMainWindow`。
- 后续可扩展暴击、治疗、护盾、免疫等样式。

### 4.6 控制模式提示

用途：

- 展示当前战斗控制状态，如自动/手动。

当前状态：

- `BattleMainWindow` 已有 `m_tmpControlMode`。

本期要求：

- 保留。
- 后续如改为图标或切换按钮，需要重新定义交互状态。

### 4.7 普通怪血量

当前状态：

- 已移除为敌方单位创建文本信息 widget 的旧实现。

设计结论：

- 普通杂兵不应在主 HUD 中逐个常驻显示，避免 1000+ 怪物时 UI 数量失控。
- 主 HUD 只显示 Boss、精英、当前目标或关键敌人。
- 普通怪如需血量表现，应使用单位头顶轻量血条或不显示。

## 7. 数据来源

### 7.1 玩家血量

策划含义：

- 展示当前战斗内玩家生命值和最大生命值。
- 玩家受到伤害、治疗、复活或服务端纠正血量时刷新。
- 显示形式为血条填充 + 数字文本。

主数据来源：

- 当前战斗中的玩家 `BattleUnit`
- `BattleStatsComponent`
- `BattleStatsComponent.Hp`
- `BattleStatsComponent.MaxHp`

兼容刷新来源：

- `BattleUnit.NumericComponent`
- `NumericType.Hp`
- `NumericType.MaxHp`

说明：

- 战斗 HUD 不直接读取主世界 `Unit` 的持久数值。
- `BattleStatsComponent` 是战斗内数值快照，战斗内扣血、治疗和服务端血量同步优先读写这里。
- `NumericComponent` 当前作为旧链路和事件刷新镜像保留，后续 UI 读取应逐步转向 `BattleStatsComponent`。

刷新入口：

- `BattleStart_UI` 打开 `BattleMainWindow` 后调用 `BattleUIHelper.OnBattleStarted`
- `BattleUnitNumericChange_BattleMainWindow`
- `BattleUnitDamaged_BattleMainWindow`
- `BattleUnitDeadView`
- `BattleUnitRevivedView`

Prefab 控制对象：

- `m_imgPlayerHp`：按 `Hp / MaxHp` 更新填充比例。
- `m_tmpPlayerHp`：显示 `Hp / MaxHp` 文本。
- `player_hp_bg`：静态血条底图，不绑定。
- `player_hp_frame`：静态血条边框，不绑定。

边界规则：

- `MaxHp <= 0` 时血条显示为空，避免除零。
- 当前血量需要限制在 `[0, MaxHp]`。
- 玩家死亡后保留 0 血显示，直到战斗结束或复活纠错。

### 7.2 Boss 血量

策划含义：

- 展示当前 Boss 或关键敌人在战斗内的生命值和最大生命值。
- Boss 血量变化必须以服务端确认数据为准。

主数据来源：

- 当前战斗中的 Boss `BattleUnit`
- `BattleStatsComponent`
- `BattleStatsComponent.Hp`
- `BattleStatsComponent.MaxHp`

服务端同步来源：

- `M2C_BossDamage`
- `M2C_Damage`
- `M2C_SyncBoss`

兼容刷新来源：

- Boss `BattleUnit.NumericComponent`
- `NumericType.Hp`
- `NumericType.MaxHp`

规则：

- Boss 血量显示以服务端确认数据为准。
- 客户端攻击 Boss 时只做预表现，不提前扣 Boss HUD 血量。
- `NumericComponent` 仅作为当前旧刷新链路镜像，不作为最终战斗 HUD 的首选读取源。

### 7.3 波次信息

来源：

- `M2C_WaveStart`
- `M2C_WaveComplete`
- `WaveStart_BattleMainWindow`
- `WaveComplete_BattleMainWindow`
- 后续如需要剩余怪物数，可由波次状态或新增消息提供。

### 7.4 发射器信息

来源待确认：

- 发射器配置
- 玩家当前拥有/启用的发射器组件
- 发射器 CD 组件或本地攻击组件
- 发射器挂载的 Buff 列表

本期不强制实现完整发射器栏数据绑定，但命名和 Prefab 结构应按发射器系统预留。

## 10. 本期范围建议

建议本期实现：

- 玩家血条 + 数字文本。
- Boss 血条区，默认隐藏，Boss 出现后显示。
- 波次文本。
- 保留控制模式文本。
- 保留伤害飘字。
- 停止将所有普通怪都作为主 HUD 敌方信息常驻展示，改为只显示 Boss/精英/当前目标。

建议延后：

- 完整发射器栏 CD。
- 多管 Boss 血条。
- Boss 阶段 UI。
- 经验条、蓝条、金币、背包位。
- 发射器拖拽、升级、锁定交互。
- Buff 挂载、卸载、替换交互。

## 11. 验收标准

- 进入战斗后玩家血条和血量文本显示正确。
- 玩家受伤或恢复时，血条和文本刷新。
- Boss 出现时显示 Boss 血条、名称和血量。
- Boss 受到伤害时，HUD 只根据服务端确认数据刷新。
- 波次开始时显示当前波次。
- 战斗结束后 HUD 清理，不残留 widget。
- 普通杂兵不会在主 HUD 中创建大量常驻信息项。
- `16:9`、`19.5:9`、平板宽屏下 HUD 不遮挡主要战斗区。
- 静态装饰节点不使用 `m_` 前缀。
- 代码控制节点使用 TEngine 生成器前缀并能在 `.g.cs` 中生成绑定字段。
