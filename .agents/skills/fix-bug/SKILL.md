# 修复功能Bug

用户问题：$ARGUMENTS

## 第零步：二次确认

判断用户描述是否属于功能Bug（代码逻辑错误、表现异常）。如果是，确认：
> 检测到你要修复「xxx」，是否启动修复流程？

## 第一步：查阅文档 + 定位问题

1. 读取 `docs/` 目录下相关的业务规范文档（如 `docs/gameplay.md`）
2. 根据文档中的设计规范，对比现有代码，定位问题根因
3. 如果问题是**文档本身过时或缺失**导致代码实现与文档不一致，先标记需要更新文档

## 第二步：确认修复方案

**必须暂停**。用 AskUserQuestion 向用户展示：

- 问题根因分析（引用文档中的规范 vs 当前代码实现）
- 修复方案（改代码 / 改文档 / 两者都改）
- 涉及的文件列表

如果文档需要修改，先让用户确认文档修改，再确认代码修复。

## 第三步：执行修复

根据修复方案类型选择执行路径：

### 路径A：仅需修改文档
- 更新 `docs/` 下的相关规范文档
- 提示用户后续需要根据新文档重新实现或修复代码

### 路径B：仅需修改代码
1. 调用 **server** 子代理 — 根据文档规范修复服务端代码
2. 调用 **client** 子代理 — 根据文档规范修复客户端代码
3. 编译验证：`dotnet build Server/Server.sln`

### 路径C：文档 + 代码都改
先执行路径A（改文档），再执行路径B（改代码）。

## 第四步：汇总

- 问题根因和修复内容
- 修改的文件列表
- 编译状态
- 如果文档有更新，标注文档变更内容
