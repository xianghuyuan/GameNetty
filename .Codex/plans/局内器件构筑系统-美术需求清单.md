## 0. 文档定位

说明：

- 本文件不以传统装备栏美术为目标
- 文中若出现“道具、物件、遗物”等词，均应优先理解为局内成长选择的视觉承载
- `Emitter`、`Buff`、`Modifier` 是程序内部执行模型，不进入美术表达
- 视觉表达优先级高于系统术语表达
- 整体气质优先级：`柔和` > `童话` > `战斗功能感`

### 0.1 与程序层的边界

美术需求只处理玩家能理解和感受到的视觉对象，不处理程序执行分类。

程序层概念：

- `Emitter`：按频率或条件触发的执行器
- `Buff`：被触发后施加到目标身上的效果
- `Modifier`：改写触发频率、目标、范围、伤害、Buff 强度等参数

这些概念不进入图标、HUD、弹窗和文案表达。  
一个玩家看到的“月灯”“玫瑰火种”“旧王冠”之类物件，程序上可能对应多个 `Emitter`、多个 `Buff` 或一个 `Modifier`，但美术不需要按这些内部分类拆图标。

美术应优先表达：

- 这是旅途中获得的小物
- 这是祝福的象征物
- 这是旧王国或圣堂留下的遗物
- 这是屠龙后的纪念物
- 这是当前局构筑路线的代表性物件

不应优先表达：

- 这是发射器
- 这是 Buff
- 这是 Modifier
- 这是固定槽位装备

## 1. 目标

第一版美术目标不是做大规模资产，而是支撑以下体验：

1. 候选物件出现时，玩家能清楚识别其视觉类型与气质
2. 玩家能快速看懂新旧成长项对比
3. 局内 HUD 能明确显示当前局内构筑状态
4. 替换旧物件后，钱币回收反馈明确

第一版关键词：

- 清晰
- 可读
- 柔和
- 童话
- 低成本
- 可扩展

## 2. 风格要求

### 2.1 整体视觉基调

本项目整体视觉基调定义为：

**奶油系轻幻想月夜冒险**

世界呈现参考蓝紫月夜童话森林、奶油色圆角 UI 和轻幻想绘本构图。  
画面不是厚重写实西幻，也不是高饱和儿童绘本，而是带有温柔月夜、轻手绘质感和清晰战斗 HUD 信息层级的童话冒险表现。  
整体应偏柔和、轻盈、亲和，像月夜里的奶油系幻想绘本，而不是强调硬朗战斗装备感。

### 2.2 核心视觉特征

#### 奶油系轻幻想

画面语言参考圆润、轻量、可读的童话游戏 UI：

- 香草色圆角面板
- 浅粉、薄荷绿、浅蓝和浅金点缀
- 柔和描边
- 低对比蓝紫阴影
- 轻微糖霜高光
- 清晰、低厚度的 HUD 组件

要求整体有“温柔童话冒险 UI”质感，而不是自然主义写实场景。  
观感应偏轻、偏软、偏故事感，不要过重、过硬、过粗粝，也不要过甜品化。

#### 柔和夜色

整体光色以安静、柔和、可长时间观看为优先。

推荐主色方向：

- 雾蓝
- 灰蓝
- 蓝紫
- 柔和松绿
- 香草奶油色
- 浅粉
- 浅金

要求：

- 不走纯黑压暗
- 不走高反差重阴影
- 不做刺眼荧光高亮
- 允许梦幻感，但必须服务可读性
- 整体优先“柔和夜色”，而不是“暗黑压迫”

#### 奶油色 UI 材质

材质应强调圆润、干净、亲和的轻幻想 HUD 质感。

推荐材质与配色：

- 香草色面板
- 奶油白
- 浅粉填充
- 薄荷绿填充
- 浅蓝灰阴影
- 浅金小装饰

要求：

- 整体非金属主导
- 避免现代工业感
- 避免强镜面反射
- 避免脏旧纸板和重木框
- 月光下的轻幻想小物件
- 温柔但不幼稚，清爽但不发白

#### 低疲劳阅读

由于游戏存在长时间推图和持续战斗，视觉必须优先考虑低疲劳阅读体验。

要求：

- 大形优先，小装饰后置
- 低频纹理，避免高密度噪点
- 细节克制，不堆花纹
- 轮廓清楚，便于无限滚动和长期观看
- UI 与战斗信息层级明确，不互相抢读

#### 童话但不幼

项目气质应是“传奇童话”，不是“儿童绘本”。

要求：

- 有梦幻感
- 有故事书舞台感
- 不使用过度鲜艳的幼态糖果色
- 不做卡通过头的表情化夸张
- 不抢战斗可读性，不削弱战斗张力
- 可以温柔，但不要幼稚；可以浪漫，但不要甜腻

### 2.3 题材来源

局内成长物件的题材包装建立在“勇者击败恶龙、拯救公主”的童话冒险主线之上，视觉与命名来源可归纳为：

- 童话遗珍
- 勇者远征
- 王国遗物
- 圣堂器具
- 屠龙战利品
- 恶龙遗骸改装件

### 2.4 避免方向

不建议风格：

- 纯赛博插件
- 纯现代枪械零件
- 实验室芯片风
- 儿童绘本高亮糖果色
- 厚重暗黑写实西幻
- 金属机甲感

目标是让这些物件看起来像“远征途中获得的童话遗物与小道具”，而不是现代装备系统。  
第一观感应是“故事里的东西”，第二观感才是“战斗中会用到的东西”。

## 4. 道具图标需求

第一版建议总量：

- 道具图标 18 个

若产能不足，可先做 9 个占位图标，后续补全。

### 4.1 气质描述

这些道具不应被画成明确的装备零件，也不应像系统化的功能部件。  
它们更像是童话舞台里被反复使用、修补、收藏和传承的物件。

有些来自远行路上的行囊与木箱，绑着布带、蜡绳和旧金属扣；  
有些像王宫和圣堂里留下来的小礼器，被放在抽屉、匣子、架子和幕布后面，沾着旧纸页、香灰和月光；  
还有一些则像屠龙之后留下的纪念物，带着骨、鳞、灰烬和封印的痕迹，但仍然属于童话故事里的东西，而不是黑暗奇幻里的诅咒遗物。

它们首先应该让人联想到：

- 故事书插图里的小物件
- 幕布后的小匣子
- 王宫角落里的旧礼器
- 月光下的手工遗物

### 4.2 视觉倾向

这些物件应当带一点“被珍藏的小物件”的温度，但不是写实脏旧。  
更像奶油系月夜童话里的冒险小物：

- 轮廓圆润
- 面色干净
- 浅金只是小面积点缀
- 可有轻微手绘高光
- 装饰不繁复，但有一点点讲究
- 像从月夜城堡、花园、远征背包里拿出的轻幻想纪念物
- 边缘圆润一些，避免过尖、过刺、过硬的形态

它们不需要强烈说明“这是攻击类”“这是增益类”“这是联动类”。  
玩家先感受到的是它来自哪里、属于什么故事、带着什么气息，功能区分交给 UI 标签和交互去补。  
不要让图标一眼看上去像“攻击模块、增益模块、联动模块”。

### 4.3 气质关键词

建议统一使用这些关键词指导美术产出：

- 奶油系轻幻想
- 故事书插图
- 月夜童话森林
- 香草色圆角面板
- 王宫小匣
- 圣堂小礼器
- 屠龙后的纪念物
- 月光下的轻幻想物件
- 梦幻但克制
- 童话，但不是儿童感
- 柔和
- 安静
- 旧舞台
- 小礼物感

### 4.4 避免方向

避免把这些物件画成：

- 明确的枪械部件
- 纯功能型装备模块
- 强工业结构件
- 太像游戏掉落物的标准图标
- 太亮、太甜、太可爱的儿童绘本道具
- 过重的武器化表达
- 过强的系统分类感

---

## 5. 品质边框需求

第一版只做 3 档：

- 普通
- 精良
- 稀有

建议表现：

- 普通：灰 / 白
- 精良：蓝 / 绿
- 稀有：紫 / 金

要求：

- 小图标尺寸下能识别
- 候选大卡片尺寸下也成立
- 与不同来源的道具图标兼容

---

## 6. 道具类型标识需求

道具类型可以区分，但区分方式应优先服务世界观包装，而不是直接暴露程序术语。

建议玩家可见分类尽量弱化，不一定需要在 UI 中强行展示总分类词。

如果必须显示，优先考虑更柔和的表达：

- 行装
- 祝物
- 遗珍

建议至少同时具备两种区分方式：

1. 颜色区分
2. 图标角标
3. 文本标签

若需要可见分类，第一版建议：

- 行装：偏暖棕、旧红、布带感
- 祝物：偏雾蓝、月白、淡金
- 遗珍：偏灰紫、旧金、暗青

## 7. 候选弹窗页面需求

### 7.1 页面用途

当候选成长项出现时弹出，战斗暂停，玩家在此界面完成接受/替换/保留决策。

### 7.2 页面结构

1. 标题区
- 文案示例：`发现旅途遗物`

2. 候选物件卡
- 图标
- 名称
- 稀有度
- 类型
- 效果描述

3. 当前路线/旧物件卡
- 图标
- 名称
- 稀有度
- 类型
- 效果描述
- 起手阶段可显示“新的旅途物件”或“尚未获得同类物件”

4. 对比区
- 新旧参数差异
- 上升/下降箭头
- 变化摘要

5. 回收区
- 替换后旧物件回收金币显示
- 起手阶段不显示

6. 按钮区
- 起手阶段：`收下`
- 正常阶段：`接受` / `保留当前`

### 7.3 页面状态

至少支持：

- 起手强制接受版
- 正常选择版
- 不同稀有度视觉变化

---

## 8. HUD 页面需求

### 8.1 页面用途

常驻显示当前远征状态与少量代表性局内成长。

建议整体理解为：

- 左侧是角色印记
- 中部是远征状态
- 右侧是当前旅途中随身携带的重要物件

不建议把 HUD 理解成传统装备栏或功能面板拼接。

### 8.2 展示内容

1. 远征行囊 / 随身物件区域
- 少量代表性物件
- 当前主要能力提示
- 当前代表性祝福或遗珍
- 当前构筑路线摘要

每个物件显示：

- 当前道具图标
- 品质边框
- 轻量占位或行囊空位提示

说明：

- 右侧区域不是完整构筑列表
- 不要求显示所有已获得成长项
- 不按 `Emitter / Buff / Modifier` 分类摆放
- 不按固定三格装备栏设计
- 它更像勇者随身行囊里露出的几件代表性物件，或当前路线的一小段视觉摘要

完整构筑详情应放在：

- 暂停面板
- 局内详情页
- 节点选择/结算界面

2. 远征记录区域
- 钱币图标
- 当前数值

### 8.3 风格要求

- 尽量轻量
- 不遮挡战斗主体
- 更偏状态条与构筑摘要，而不是大面板
- 更像舞台边缘的一条远征记录栏，而不是传统装备 UI
- 不做传统三槽装备栏，不要求三个固定方格

## 9. 钱币资源表现需求

第一版钱币虽然不消费，但必须有存在感。

需求包括：

1. 钱币图标
2. 替换后回收提示
3. HUD 钱币数字刷新表现

建议风格：

- 王国铸币
- 屠龙战利金
- 勇者远征金币

避免现代商城币感。

候选弹窗需要最基础的对比符号。

需要：

1. 玩家能一眼分辨道具属于哪一类视觉来源

## 16. 第一版结论

第一版美术重点不是堆量，而是服务清晰决策：

> 玩家在推图中停下来做一次成长选择时，必须能快速看懂“这是什么、会带来什么变化、旧物件会换来什么”。
